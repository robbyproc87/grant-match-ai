## Database schema (Phase 1)
Tables — apply Supabase RLS on every one (only org members can read/write their org's data):
- `orgs` (id, name, ein, owner_user_id, created_at)
- `org_members` (org_id, user_id, role: owner|admin|writer|viewer)
- `org_profiles` (org_id PK, mission, focus_areas[], populations_served[], geographies[], annual_budget, years_operating, past_grants_received jsonb, updated_at)
- `programs` (id, org_id, name, description, target_population, geography, budget, outcomes_summary)
- `funders` (id, ein UNIQUE, name, type, website, propublica_data jsonb, last_synced_at)
- `grants` (id, funder_id, name, amount_min, amount_max, deadline, deadline_type: rolling|fixed|loi, eligibility jsonb, focus_areas[], geographies[], source: grants_gov|propublica|manual, source_url)
- `applications` (id, org_id, grant_id, stage: researching|drafting|submitted|decided|awarded|declined, created_at, updated_at)
- `match_scores` (org_id, grant_id composite PK, score_total, score_breakdown jsonb, computed_at)
- `messages` (id, application_id NULLABLE, org_id, user_id, role: user|assistant, content, created_at)

## Match scoring algorithm v1 (compute server-side, cache in `match_scores`)
score_total (0-100) = sum of:
- **mission_alignment (0-30)**: semantic similarity org mission/focus_areas ↔ grant focus_areas. Single Haiku call returns score with reasoning. Cache.
- **geographic_fit (0-15)**: 15 if org geographies ∩ grant geographies non-empty OR grant is national; else 0
- **budget_fit (0-15)**: 15 if org annual_budget within funder's typical range OR funder unrestricted; 5 if close; 0 if mismatch
- **eligibility (0-15)**: binary check against grant.eligibility JSON (501(c)(3), org age, etc.)
- **population_alignment (0-15)**: semantic similarity org populations_served ↔ grant target populations. Haiku call.
- **prior_relationship (0-10)**: +5 prior application with this funder, +10 prior award

Return total + full factor breakdown. UI shows total, expandable to show factors. Recompute on org_profile or grant change.

## ProPublica integration
- API: https://projects.propublica.org/nonprofits/api (free, no key needed, rate-limited — cache aggressively)
- Build `lib/sources/propublica.ts` implementing a `SourceAdapter` interface so Phase 2 sources (Grants.gov, Candid, etc.) plug in without rewrite
- Adapter contract: `searchFoundations(query)`, `getFoundationDetails(ein)`, `getRecentGrants(ein)`
- On first run, seed `funders` table with ~25 foundations matching keywords: "pediatric", "children's health", "Arizona", "hospital", "rare disease"
- Each funder row should populate: recent grants (3 yrs), avg grant size, board info when available

## Claude chat integration
- Route: `app/api/chat/route.ts`, streaming response (Vercel AI SDK or native stream)
- System prompt must include: org name, mission, focus areas, annual budget, populations served, current grant context (if user is viewing one)
- Model: `claude-sonnet-4-6`
- Persist user + assistant messages to `messages` table
- No tool use yet — contextual chat only
